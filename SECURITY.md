# Security Policy

## Reporting a Vulnerability
Please do not open public issues for security vulnerabilities.

Preferred path:
- Use GitHub Security Advisories (private report) for this repository.

Include as much detail as possible:
- affected component/path
- impact
- reproduction steps or proof of concept
- suggested mitigation if available

## Disclosure Process
- We acknowledge reports as quickly as possible.
- We validate impact and scope.
- We prepare and release a fix.
- We coordinate disclosure timing with the reporter when relevant.

## Supported Versions
This project is actively developed on `main`. Security fixes are applied to maintained code in current development.
